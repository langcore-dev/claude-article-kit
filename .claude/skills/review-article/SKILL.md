# 記事レビュースキル

記事ドラフトを読み込み、article-template.md の文章ルールとレビュー観点に基づいてフィードバックを行うスキル。レビュー結果をユーザーに提示し、承認後に修正を適用する。

## 前提ファイル

このスキルは以下のファイルと連携する。すべてプロジェクトルートからの相対パスで参照すること。

- `.claude/skills/write-article/article-template.md` - 記事テンプレートと文章ルール
- `.claude/skills/review-article/review-checklist.md` - レビュー観点チェックリスト

## ワークフロー

### Phase 1: 記事の読み込み

AskUserQuestion ツールを使い、レビュー対象を特定する。

**質問1: 対象記事の指定**

```
question: "レビューする記事を指定してください"
options:
  - label: "ファイルパスを指定"
    description: "articles/ ディレクトリ内の記事ファイルパスを教えてください"
  - label: "最新の記事"
    description: "articles/ ディレクトリ内で最も新しい記事をレビューする"
```

- 「最新の記事」の場合: `articles/` ディレクトリを Glob で検索し、最新のファイルを自動選択する
- 記事ファイルを Read ツールで読み込む

**質問2: レビューの重点（任意）**

```
question: "特に重点的にレビューしたい観点はありますか？"
options:
  - label: "全体的にレビュー"
    description: "すべての観点でバランスよくレビューする"
  - label: "読みやすさ重視"
    description: "文章の流れ、リズム、わかりやすさを重点的にチェック"
  - label: "訴求力重視"
    description: "候補者や読者への響き方、魅力の伝わり方を重点的にチェック"
  - label: "正確性重視"
    description: "事実関係、技術用語、表記の正確さを重点的にチェック"
```

### Phase 2: レビュー実行

`.claude/skills/review-article/review-checklist.md` と `.claude/skills/write-article/article-template.md` を読み込み、以下の観点でレビューを行う。

#### レビュー観点

| カテゴリ | チェック項目 |
|---|---|
| **構成** | TL;DR が記事の要約として機能しているか / セクション間の流れが自然か / 各セクションに「」引用があるか |
| **文章ルール** | ですます調で統一されているか / 一文100文字以内か / 助詞の連続がないか / 冗長表現がないか |
| **視点の一貫性** | 一人称/三人称が混在していないか / 著者の発言と他者の発言の扱いが正しいか |
| **具体性** | 固有名詞・数値・技術名が適切に使われているか / エピソードが具体的か |
| **訴求力** | タイトルにフックがあるか / 読者が「読みたい」と思える導入か / メッセージが明確か |
| **textlint** | textlint を実行してエラーがないか |

#### レビュー結果の構造

レビュー結果を以下の形式でまとめる。

```
## レビューサマリー

**総合評価**: {A / B / C}
- A: 公開可能な品質。軽微な修正のみ
- B: 良い内容だが改善の余地あり。修正推奨
- C: 構成や内容に大きな課題あり。書き直し推奨

**良い点**:
- {具体的な良い点を2〜3個}

**改善点**:
- {カテゴリ}: {具体的な指摘と改善案}
- {カテゴリ}: {具体的な指摘と改善案}

**textlint 結果**: {エラー数}件
```

### Phase 3: フィードバック提示

レビュー結果をユーザーにテキストで提示し、AskUserQuestion で対応方針を確認する。

```
question: "レビュー結果を確認してください。どう対応しますか？"
options:
  - label: "すべて修正する"
    description: "指摘事項をすべて反映する"
  - label: "選んで修正する"
    description: "修正する項目を選択する"
  - label: "修正しない"
    description: "現状のままで確定する"
```

### Phase 4: 修正の適用

- 「すべて修正する」→ すべての改善点を Edit ツールで適用し、textlint を再実行する
- 「選んで修正する」→ ユーザーに修正する項目を選んでもらい、選択された項目のみ適用する
- 「修正しない」→ 完了メッセージを表示する

修正後は記事全文を Read ツールで読み込み、ユーザーに提示する。
