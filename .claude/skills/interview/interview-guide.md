# インタビュー質問ガイド

## 質問カテゴリ

AI はこの5カテゴリからバランスよく質問を動的に生成する。各カテゴリのシード質問は参考例であり、文脈に応じて自由にアレンジすること。

### 1. 技術・プロダクト

対象者の技術力と課題解決力を引き出す。

- 現在担当しているプロダクトやサービスについて教えてください
- チームで使っている技術スタックを教えてください
- 最近取り組んだ技術的に難しかった課題は何ですか
- 技術選定で意識していることはありますか
- コードレビューやアーキテクチャ設計でこだわっている点はありますか

### 2. チーム・文化

チームの雰囲気や働く環境の魅力を伝える。

- チームの雰囲気を一言で表すとどんな感じですか
- チーム内のコミュニケーションはどのように行っていますか
- 技術的な意思決定はどのようなプロセスで行われますか
- チームで大切にしている価値観やルールはありますか
- 他のチームとの連携はどのように行っていますか

### 3. キャリア・成長

入社動機や成長ストーリーで候補者の共感を得る。

- この会社に入社した決め手は何でしたか
- 入社前と入社後でギャップはありましたか
- エンジニアとして最も成長を実感した経験を教えてください
- スキルアップのために取り組んでいることはありますか
- メンターや上司からのサポート体制はどうですか

### 4. 日常・働き方

候補者が入社後の生活をイメージできるようにする。

- 典型的な1日のスケジュールを教えてください
- リモートワークと出社のバランスはどうですか
- 開発プロセス（スプリント、デプロイ頻度等）を教えてください
- オンボーディングの体験はどうでしたか
- ワークライフバランスについてどう感じていますか

### 5. 未来・ビジョン

会社やチームの将来性を示し、候補者のモチベーションに訴求する。

- 今後チャレンジしたい技術領域や目標はありますか
- チームやプロダクトの今後の方向性を教えてください
- この会社で実現したいことは何ですか
- 一緒に働きたいエンジニア像はどんな人ですか
- 入社を検討している方へメッセージをお願いします

## 質問選択アルゴリズム

1. Phase 1 で取得した「記事の目的」「ターゲットペルソナ」「対象者情報」を踏まえ、最も効果的なカテゴリから開始する
2. 各カテゴリから最低1問は出題し、全体で5〜8問を目安とする
3. 前の回答の内容に応じて次の質問を選定する（例: 技術的な話題が盛り上がったら技術カテゴリをもう1問追加）
4. 最後の質問は必ず「未来・ビジョン」カテゴリから選び、前向きな締めくくりにする

## 深掘りルール

### トリガー条件

以下のいずれかに該当する場合、深掘り質問を追加する。

| 条件 | 判定基準 | 深掘りアクション |
|---|---|---|
| 短すぎる回答 | 回答が30文字未満 | 「もう少し詳しく教えていただけますか？」と具体例を求める |
| 抽象的な表現 | 「良い」「楽しい」「すごい」「面白い」「充実」等の形容詞のみ | 「具体的なエピソードや場面を教えてください」と事例を求める |
| 数値・固有名詞の欠如 | 技術質問なのに具体的な技術名や数値が出ない | 「具体的な技術名やツール名を教えてください」と特定を促す |

### 制約

- 同一質問に対する深掘りは最大2回まで
- 深掘り後も具体性が不足する場合は次の質問に移る
- 深掘り質問はプレッシャーを与えない柔らかい口調にする

## 行動経済学を意識した質問設計

インタビューの中で、記事執筆時に行動経済学テクニックを活用できる素材を意識的に引き出す。各カテゴリの質問生成時に、以下のテクニックに対応する素材が得られるよう質問を設計する。

| テクニック | 引き出すべき素材 | 質問の方向性 |
|---|---|---|
| アンカリング | インパクトのある数値・成果 | 「数字で表すとどのくらいですか？」「改善幅はどのくらいでしたか？」 |
| 社会的証明 | 周囲の反応・チームの同調 | 「チームメンバーの反応はどうでしたか？」「他のチームにも広がりましたか？」 |
| 損失回避 | 環境の希少性・機会の貴重さ | 「この経験は御社ならではだと感じますか？」「このタイミングだからこそできたことは？」 |
| フレーミング | 課題のポジティブな側面 | 「大変だった中で、逆に面白かった部分はありますか？」 |
| ピーク・エンド | 最も感情が動いた瞬間 | 「一番印象に残っている瞬間は？」「一番達成感を感じた場面は？」 |
| 物語性 | 時系列の変化・転機 | 「入社前→入社後で一番変わったことは？」「転機になった出来事は？」 |
| 内集団バイアス | チームの共通価値観・仲間意識 | 「チームの一員として誇りに思うことは？」「どんな人と一緒に働きたいですか？」 |

### カテゴリ別の活用ポイント

- **技術・プロダクト**: アンカリング（数値成果）、フレーミング（技術的課題の面白さ）
- **チーム・文化**: 社会的証明（チームの共通行動）、内集団バイアス（仲間意識）
- **キャリア・成長**: 物語性（キャリアのストーリーアーク）、ピーク・エンド（転機）
- **日常・働き方**: 認知容易性（具体的な日常のイメージ）、社会的証明（働き方の満足度）
- **未来・ビジョン**: 損失回避（今しかない機会）、内集団バイアス（未来の仲間への呼びかけ）
