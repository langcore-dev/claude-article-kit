# 採用広報インタビュースキル

自社エンジニアに対して対話形式でインタビューを行い、収集した一次情報から採用広報記事のドラフトを生成するスキル。

## 前提ファイル

このスキルは以下のファイルと連携する。すべてプロジェクトルートからの相対パスで参照すること。

- `.claude/skills/interview/interview-guide.md` - 質問カテゴリと深掘りルール
- `.claude/skills/interview/article-template.md` - 記事テンプレートと文章ルール
- `.claude/agents/article-writer.md` - 記事生成サブエージェント

## ワークフロー

### Phase 1: コンテキスト初期化

AskUserQuestion ツールを使い、以下の情報を収集する。

**質問1: 記事の目的とターゲット**

```
question: "この記事の目的とターゲットを教えてください"
options:
  - label: "エンジニア採用（新卒向け）"
    description: "新卒エンジニア候補者に向けた社員インタビュー記事"
  - label: "エンジニア採用（中途向け）"
    description: "中途エンジニア候補者に向けた社員インタビュー記事"
  - label: "技術ブランディング"
    description: "技術力の高さをアピールするインタビュー記事"
  - label: "チーム紹介"
    description: "チームの雰囲気や文化を伝えるインタビュー記事"
```

**質問2: 対象者の基本情報**

AskUserQuestion で以下を自由入力で取得する。

```
question: "インタビュー対象者の情報を教えてください（名前、役職、チーム名、入社時期）"
```

回答例: 「山田太郎、バックエンドエンジニア、プラットフォームチーム、2023年4月入社」

取得した情報を以下の形式で内部的に保持する。

```
context:
  purpose: {記事の目的}
  persona: {ターゲットペルソナ}
  profile:
    name: {名前}
    role: {役職}
    team: {チーム名}
    joined: {入社時期}
```

### Phase 2: インタビュー実行

`.claude/skills/interview/interview-guide.md` を読み込み、質問カテゴリと深掘りルールを把握する。

#### 質問の進め方

1. Phase 1 のコンテキストに基づき、最適な最初の質問を生成する
2. AskUserQuestion を使い、1問ずつ質問する
   - question に質問文を設定する
   - options には使わず、ユーザーが「Other」から自由入力で回答する形式にする
   - ただし回答の参考になる選択肢を2〜3個用意してもよい
3. 回答を受け取ったら、深掘りルールに照らして判定する
4. 深掘りが必要な場合は追加質問を行う（最大2回）
5. 次の質問を前の回答の内容を踏まえて動的に生成する
6. 全5カテゴリからバランスよく、合計5〜8問を実施する
7. 最後の質問は「未来・ビジョン」カテゴリから選ぶ

#### 質問生成のルール

- シード質問をそのまま使わず、対象者のプロフィールや前の回答に合わせてカスタマイズする
- 質問は自然な会話口調にする（「〜について教えてください」「〜はいかがですか」）
- 前の回答で出たキーワードを拾って次の質問に織り込む
- 対象者が答えやすいよう、具体的な切り口を提示する

#### 深掘りの判定

interview-guide.md の深掘りルールに従う。

- 回答が30文字未満 → 具体例を求める
- 抽象的な表現のみ → エピソードを求める
- 技術質問で具体名なし → 技術名・ツール名を求める
- 深掘りは同一質問で最大2回。それでも具体性が不足すれば次の質問へ進む

#### Q&A データの蓄積

各質問と回答を以下の形式で蓄積する。

```
qa_log:
  - category: {カテゴリ名}
    question: {質問文}
    answer: {回答文}
    followups:
      - question: {深掘り質問}
        answer: {深掘り回答}
```

### Phase 3: ドラフト生成

インタビューが完了したら、記事生成をサブエージェントに委任する。

#### サブエージェントの起動

Task ツールを使い、`article-writer` サブエージェントを起動する。プロンプトに以下を含める。

```
以下のインタビューデータから採用広報記事を作成してください。

## 記事の目的
{purpose}

## ターゲットペルソナ
{persona}

## 対象者プロフィール
- 名前: {name}
- 役職: {role}
- チーム: {team}
- 入社時期: {joined}

## インタビュー記録
{qa_log の全内容をテキストとして展開}

## 出力先
{プロジェクトルート}/articles/{YYYY-MM-DD}-{名前のローマ字}.md

## 指示
1. `.claude/skills/interview/article-template.md` を読み、テンプレートと文章ルールに従って記事を作成してください
2. article-template.md の「行動経済学テクニック」セクションを参照し、素材に合ったテクニックを自然に組み込んでください（アンカリング、社会的証明、損失回避、フレーミング、ピーク・エンドの法則、物語への没入、認知容易性、内集団バイアス）
3. 記事を出力先パスに書き出してください
4. `npx textlint {出力先パス}` を実行し、エラーがあれば修正してください
5. textlint エラーが0になるまで修正を繰り返してください
```

### Phase 4: レビュー・仕上げ

サブエージェントの完了後、以下を行う。

1. 生成された記事ファイルを Read ツールで読み込む
2. ユーザーに記事全文を提示する
3. AskUserQuestion で修正の要否を確認する

```
question: "記事のドラフトが完成しました。修正したい箇所はありますか？"
options:
  - label: "このままでOK"
    description: "修正不要、記事を確定する"
  - label: "部分的に修正したい"
    description: "特定のセクションや表現を修正する"
  - label: "全体的に書き直したい"
    description: "トーンや構成を含めて大幅に修正する"
```

- 「このままでOK」→ 完了メッセージを表示
- 「部分的に修正したい」→ 修正箇所を聞いて Edit ツールで対応
- 「全体的に書き直したい」→ フィードバックを聞いてサブエージェントを再起動
